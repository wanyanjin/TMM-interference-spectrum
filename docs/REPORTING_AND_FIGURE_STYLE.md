# REPORTING_AND_FIGURE_STYLE.md

本文档定义 `TMM-interference-spectrum` 项目的正式报告、审计报告、CLI 输出图、组会图和论文草图的统一交付规范。

---

## 1. Markdown 报告规范

### 1.1 基本要求

1. 正式报告必须有执行摘要。
2. 核心结论必须前置。
3. 数学公式必须使用 LaTeX，不得用普通 text code block 表达公式。
4. 普通 code block 只用于命令、路径、JSON、CSV 字段、终端输出。
5. 超过 3 个关键数值时，优先使用表格。
6. 复杂数结果应表格化，至少包含：
   - `real`
   - `imag`
   - `magnitude`
   - `phase_rad`
   - `phase_deg`
   - `reflectance`，如适用
7. 所有反射率必须标明是 `0–1` 还是 `%`；本项目默认使用 `0–1`。
8. 报告中的文件路径默认使用相对路径。
9. 绝对路径只允许出现在附录或机器可读 JSON 中。
10. 报告必须列出假设、风险和不能证明的内容。
11. 审计报告必须包含 `PASS / WARNING / FAIL / NOT_CHECKED` 状态表。

### 1.2 推荐正式报告结构

```md
# Report Title

## 1. 执行摘要
## 2. 输入数据与版本
## 3. 方法与公式
## 4. 关键结果
## 5. 质量控制
## 6. 讨论
## 7. 风险与假设
## 8. 下一步建议
## 9. 附录
```

单波长 trace 报告允许使用更贴近审计链路的章节结构，但也必须包含：

- 执行摘要
- 方法与公式
- 关键结果
- 状态检查表
- 风险与不能证明的内容

## 2. LaTeX 公式规范

### 2.1 行内公式

```md
$R(\lambda)$
```

### 2.2 块级公式

```md
$$
R_{\mathrm{exp}}(\lambda)
=
\frac{I_{\mathrm{sample}}(\lambda)/t_{\mathrm{sample}}}
{I_{\mathrm{ref}}(\lambda)/t_{\mathrm{ref}}}
\cdot R_{\mathrm{ref}}(\lambda)
$$
```

### 2.3 必须使用 LaTeX 的公式类型

- 曝光归一化
- 反射率校准
- Fresnel 系数
- 相干薄膜相位
- 单层薄膜振幅反射
- 厚玻璃非相干级联
- 残差
- 相对误差
- mask / QC 指标，如适用

## 3. 可视化图片规范

### 3.1 默认导出参数

正式展示图默认采用：

- `aspect ratio = 3:2`
- `figsize = 7.2 × 4.8 inch`
- `dpi = 300`
- `background = white`

输出格式：

- `PNG`：用于 PPT / 组会
- `PDF` 或 `SVG`：用于论文草图 / 矢量编辑

### 3.2 通用要求

1. 默认图例放在图内，不放在图外。
2. 图例不得遮挡关键数据。
3. 同一类图必须保持相同坐标轴范围。
4. 坐标轴必须包含单位。
5. 同一物理含义必须使用一致颜色和线型。
6. 不得通过自动缩放误导比较。
7. 默认不要用 `bbox_inches="tight"` 改变最终比例，除非确认导出比例仍符合规范。
8. 图中文字必须清晰，适合直接放入 PPT。
9. 每张正式图必须能追溯到输入数据、脚本、manifest 或报告。
10. 输出文件名必须包含 phase、数据集、图类型、波段或关键参数。

### 3.3 推荐字体与样式

- `font: Arial / DejaVu Sans / Helvetica`
- `title: 12–14 pt`
- `axis label: 11–12 pt`
- `tick label: 9–10 pt`
- `legend: 9–10 pt`
- `line width: 1.8–2.2`
- `marker size: 3–5`

### 3.4 推荐图例语义

- 实验 `glass/Ag` 链：`solid line`
- 实验 `Ag mirror` 链：`solid line`，不同颜色
- `TMM fixed`：`dashed line`
- `TMM best-d`：`dash-dot line`
- 残差：`solid line`
- invalid / masked points：`light gray`，低 `alpha`

### 3.5 推荐坐标范围

- 主反射率对比：`x = 400–750 nm`, `y = 0–1`
- 重点可见区：`x = 500–750 nm`, `y = 0–0.6`
- Ag 参比反射率：`x = 400–750 nm`, `y = 0.8–1.05`
- 残差：`x = 400–750 nm`, `y` 固定，例如 `-0.3–0.3`，或在图注说明

允许例外，但必须在图生成代码或报告中说明原因。

## 4. 图类型模板

| 图类型 | 推荐比例 | 默认 x 轴 | 默认 y 轴 | legend |
| --- | --- | --- | --- | --- |
| 反射率对比 | 3:2 | 400–750 nm | 0–1 | 图内 |
| 局部反射率对比 | 3:2 | 500–750 nm | 0–0.6 | 图内 |
| 残差图 | 3:2 | 400–750 nm | 固定或对称范围 | 图内 |
| counts/ms | 3:2 | 400–750 nm | 同批固定 | 图内 |
| thickness scan | 3:2 | 厚度范围 | cost | 图内 |
| heatmap | 1:1 或 4:3 | 视具体情况 | 视具体情况 | colorbar 固定 |
| 多面板图 | 4:3 或 16:9 | 视具体情况 | 视具体情况 | 统一 |

## 5. 正式交付前检查

- [ ] 公式使用 LaTeX
- [ ] 反射率单位写明为 `0–1` 或 `%`
- [ ] 文件路径为相对路径
- [ ] 关键结果有执行摘要
- [ ] 数值结果表格化
- [ ] 图比例符合规范
- [ ] 图例在图内且不遮挡关键数据
- [ ] 坐标轴单位完整
- [ ] 同类图坐标轴范围一致
- [ ] PNG 和 PDF/SVG 按需输出
- [ ] 报告列出假设、风险和不能证明的内容
- [ ] 输出能追溯到输入文件、manifest 和脚本
