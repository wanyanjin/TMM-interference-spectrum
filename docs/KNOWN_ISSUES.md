# KNOWN_ISSUES.md

本文档记录当前可见的结构债务、科学风险与环境复现风险，并单列“待核对”事项。

---

## 1. 结构债务

1. `data/raw/` 尚未成为统一原始数据入口，`test_data/` 仍承担部分原始样本/夹具职责。
2. 文献原文目录当前主要在 `reference/`，与“目标规范目录 `resources/references/`”存在偏差。
3. 根目录缺少 `README.md`，新成员环境恢复与流程入口不够清晰。
4. 仓库存在 `.DS_Store`、`__pycache__` 等非核心产物，需要在后续清理与忽略规则中收敛。
5. `src/scripts/` 中仍有大量“分析 + 建模 + 出图 + 报告”耦合脚本，薄脚本与核心模块边界有待进一步拆分。
6. 当前尚未建立统一 CLI 文档索引与完整 CLI 详细说明集。
7. 现有 `src/scripts/stepXX_*.py` 中部分高频脚本仍是 script-only，后续应逐步提升为正式 CLI。
8. CLI 参数 schema 与 GUI 参数模型尚未统一。
9. 过去部分正式报告曾用普通 text code block 表达公式，展示质量与可审查性不足。
10. 过去部分图的比例、legend 位置与坐标轴范围不统一，不利于组会和论文草图复用。
11. 现已新增报告/绘图规范；后续新报告与新图需遵守，旧图是否重绘需另立任务处理。

---

## 2. 科学与建模风险

1. 材料 `n/k` 在测量窗口外的外推仍是敏感点，需要持续标注适用范围与不确定性。
2. 参比模型存在语义差异（`glass/Ag`、`Ag mirror` 等），必须显式区分，避免混用。
3. 反演参数贴边风险需持续跟踪，避免把数值收敛直接当作物理可信。
4. 双窗拟合与专题扫描结果需与实验约束持续闭环，防止仅在理论沙盒内自洽。

---

## 3. 环境与复现风险

1. 当前仅有 `requirements.txt`，未固定 Python 版本，依赖版本未锁定。
2. 部分脚本历史上使用过本机/云盘路径约定，跨平台运行有潜在路径差异风险。
3. 正式自动化测试体系尚未建立（无统一 `tests/` 回归墙）。

---

## 4. 待核对（不进入确定性历史）

以下条目暂不写入 `CHANGELOG_DIGEST.md` 的确定性历史描述，待后续通过额外证据核对：

1. 某些 `PROJECT_STATE.md` 中给出的具体数值结论（如特定 `chi-square`、噪声阈值比较）是否与当前结果文件逐一一一对应。
2. 部分历史汇报资产与脚本版本是否严格同步（同名阶段内可能存在多次迭代导出）。
3. 一些阶段命名（数字 Phase 与 A-D 专题）在时间线上的映射关系是否已完全规范化。
4. 旧脚本仍未完全迁移到正式工具单元结构。
5. `src/core` 旧模块可能暂未完全满足新的 core 沙箱规则。
6. 尚未实现真正的 CSV/H5/Zarr reader adapter。
7. 尚未实现 `reflectance_qc` GUI。
8. 尚未建立完整 GUI 测试策略。
9. `requirements.txt` 仍未锁版本。
